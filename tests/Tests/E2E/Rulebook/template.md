# E2E Rulebook — Rule template

The shape every Rule in `rules.md` must follow. Behavioral header ending in the `→` result arrow; exactly one
`**Why:**` bullet. Anything else a Rule needs to say is a plain prose line under it, never another bold-label bullet.

### <flow> <given some input> → <observable end state>
- **Why:** <the user-visible behavior this proves>
