# Wire Rulebook — Rule template

The shape every Rule in `rules.md` must follow. Behavioral header ending in the `→` response contract; exactly
one `**Why:**` bullet. Anything else a Rule needs to say is a plain prose line under it, never another bold-label bullet.

### <method> <route> <given> → <response contract>
- **Why:** <the routing/serialization/streaming guarantee this protects>
