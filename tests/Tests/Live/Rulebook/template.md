# Live Rulebook — Rule template

The shape every Rule in `rules.md` must follow. Behavioral header ending in the `→` real-world effect; exactly
one `**Why:**` bullet. Anything else a Rule needs to say is a plain prose line under it, never another bold-label bullet.

### <agent/flow> <given a real prompt> → <real-world effect>
- **Why:** <the live behavior no scripted provider can prove>
