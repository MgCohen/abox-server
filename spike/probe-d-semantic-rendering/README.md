# Probe D ‚Äî semantic-model type rendering (the naming fix)

> Isolated, runnable proof under `spike/probe-d-semantic-rendering/`. Builds + runs on net10.0,
> references no repo project, reuses `Microsoft.CodeAnalysis.CSharp 4.11.0` (same as `spike/gen`,
> probe-a). Evidence: `evidence/run-output.txt`; emitted artifacts: `out/`.

## Idea

Render recipe field types via Roslyn **`ITypeSymbol.ToDisplayString(SymbolDisplayFormat)`** instead
of reflection **`Type.FullName`**. One renderer, two formats ‚Äî an **idiomatic** style and a
**fully-qualified** style ‚Äî plus a `using` set **derived from the symbols**, not from string surgery.

## Why

The core spike renders types via reflection `Type.FullName`. That path is wrong for almost every
non-trivial type, which is exactly why an alias dictionary (`int`‚Üí`System.Int32`, ‚Ä¶) was hand-rolled
and then **dropped as a mess**:

| Type the author means | `Type.FullName` gives | want (idiomatic) |
|---|---|---|
| `int` | `System.Int32` | `int` |
| `Outer.Inner` (nested) | `‚Ä¶Outer+Inner` (the `+`) | `Outer.Inner` |
| `List<int>` | ``‚Ä¶List`1[[System.Int32, ‚Ä¶]]`` (backticks + assembly-qualified) | `List<int>` |
| `Dictionary<string,Guid>` | ``‚Ä¶Dictionary`2[[‚Ä¶],[‚Ä¶]]`` | `Dictionary<string, Guid>` |
| `int?` | ``Nullable`1[[System.Int32,‚Ä¶]]`` | `int?` |
| `string?` | `System.String` (no nullability at all) | `string?` |
| `(int Count, string Name)` | ``ValueTuple`2[‚Ä¶]`` (names lost) | `(int Count, string Name)` |

The claim to prove: **the semantic model makes correct rendering free, in both styles, and lets the
`using` set be derived from symbols** ‚Äî closing `README.md` ¬ß8 **#16 "using-derivation"** and
removing the reason `Of<T>` had to render fully-qualified. It serves the invariant *"source-gen
back-fills inline / owned output"*: the emitted `.cs` must be idiomatic, committable, owned code.

## What was proved

The sample is authored as a typed recipe (`Sample.Recipe`, a `RecordRecipe` of `Field`s) whose
fields hit every hard case. The renderer resolves each `TypeRef` to a real `ITypeSymbol` (compiling
it against the support source, the probe-a trick) and renders it two ways. **Both emitted variants
compile** ‚Äî checked in-memory by Roslyn (`PROBE D: PASS`) *and* re-verified as real on-disk
`dotnet build` projects under `TreatWarningsAsErrors`.

Per-field rendered output (real, from `evidence/run-output.txt`):

| field | recipe text | idiomatic | fully-qualified |
|---|---|---|---|
| Id | `int` | `int` | `int` |
| Label | `string` | `string` | `string` |
| Key | `System.Guid` | `Guid` | `global::System.Guid` |
| Marker | `‚Ä¶Outer.Inner` (nested) | `Outer.Inner` | `global::ProbeD.Sample.Support.Outer.Inner` |
| Scores | `‚Ä¶List<int>` | `List<int>` | `global::System.Collections.Generic.List<int>` |
| Lookup | `‚Ä¶Dictionary<string, System.Guid>` | `Dictionary<string, Guid>` | `global::‚Ä¶Dictionary<string, global::System.Guid>` |
| Age | `int?` | `int?` | `int?` |
| Nickname | `string?` | `string?` | `string?` |
| Summary | `(int Count, string Name)` | `(int Count, string Name)` | `(int Count, string Name)` |

Emitted **idiomatic** record (`out/Customer.Idiomatic.cs`) ‚Äî note the **derived usings**:

```csharp
// <auto-generated/> ‚Äî emitted by probe-d RecordEmitter.
#nullable enable
using ProbeD.Sample.Support;
using System;
using System.Collections.Generic;

namespace ProbeD.Sample;

public record Customer(int Id, string Label, Guid Key, Outer.Inner Marker, List<int> Scores, Dictionary<string, Guid> Lookup, int? Age, string? Nickname, (int Count, string Name) Summary);
```

Emitted **fully-qualified** record (`out/Customer.FullyQualified.cs`) ‚Äî `global::‚Ä¶`, **no usings**:

```csharp
// <auto-generated/> ‚Äî emitted by probe-d RecordEmitter.
#nullable enable
namespace ProbeD.Sample;

public record Customer(int Id, string Label, global::System.Guid Key, global::ProbeD.Sample.Support.Outer.Inner Marker, global::System.Collections.Generic.List<int> Scores, global::System.Collections.Generic.Dictionary<string, global::System.Guid> Lookup, int? Age, string? Nickname, (int Count, string Name) Summary);
```

The `using` set (`ProbeD.Sample.Support`, `System`, `System.Collections.Generic`) is collected by
**walking the symbols** ‚Äî the type, its generic arguments, and tuple element types recursively ‚Äî and
taking each named type's containing namespace. No parsing of the rendered string.

**Settled findings:**
- `SymbolDisplayMiscellaneousOptions.UseSpecialTypes` gives the keyword aliases (`int`, `string`) free.
- `NameAndContainingTypes` renders nested types with `.` (`Outer.Inner`), never the reflection `+`.
- Generics and tuples (with element names) render correctly with no special handling.
- `using`-derivation works straight off the symbol graph ‚Äî **#16 is closeable**.
- `FullyQualifiedFormat` is genuinely self-contained: zero usings, always resolves.

## Authoring impact (the meta-question)

**Adopting the semantic model changed only the BACKEND renderer. The recipe is authored exactly as
before.** The author writes the same typed tree of records (`RecordRecipe` ‚Üí `Field` ‚Üí `TypeRef`)
the spike already uses; a field still names its type the way the author already names types. Nothing
in `Recipe.cs` or `Sample.cs` mentions Roslyn, `ITypeSymbol`, or `SymbolDisplayFormat` ‚Äî the entire
semantic-model machinery lives behind `SemanticRenderer` / `RecordEmitter`. **Same authoring shape,
strictly better output.** Choosing idiomatic-vs-fully-qualified is an **emit setting**, not an
authoring decision (cf. probe-c's `EmitTarget`): the *same recipe* lowers to either variant.

One honest nuance, and it is a backend choice not an authoring one: a `TypeRef` is C# **type text**
(`"List<int>"`, `"int?"`, `"(int Count, string Name)"`), resolved by compiling it against the
project's real types. That is the spike's existing `TypeRef` notion (¬ß8 #17 ‚Äî a type named at the
value level, not as a generic `<T>`), so it does **not** impose a new shape. It is in fact *more*
expressive than the old reflection `typeof(...)` / `Of<T>` path, which cannot even spell `int?`,
`string?`, or a named tuple. So the move is purely additive to authoring: the author gains the
ability to name types the CLR `Type` couldn't express, and loses nothing.

## How to run

```
cd spike/probe-d-semantic-rendering/src
dotnet run            # renders both styles, derives usings, writes out/*.cs, compile-checks both
```

Exit 0 + `PROBE D: PASS` means both emitted variants compiled. Output paths are anchored on
`CallerFilePath`, so `out/` always lands inside the probe folder regardless of where build artifacts
go (the `artifacts/` leak the spike already hit ‚Äî see probe-c).

## Honest limitations

- **Nullable-reference annotation needed a manual re-apply.** `GetTypeInfo` on a field-type
  expression surfaces a reference symbol as `NullableAnnotation.None`, so `string?` first rendered as
  `string` (the `?` dropped). Fixed by detecting the `NullableTypeSyntax` over a reference type and
  calling `WithNullableAnnotation(Annotated)`. A real impl wiring symbols from a *declared* member
  (not a re-parsed type expression) would carry the annotation already; for a probe that resolves
  text, this re-apply is the honest cost. Nullable **value** types (`int?` = `Nullable<int>`) are
  unaffected.
- **Reference resolution uses the running runtime's TPA list** (`Net.cs`), not a pinned reference
  pack. Fine for a probe; a real lowering should resolve against the *target* project's compilation
  so types it owns (and sibling-generated types) resolve ‚Äî which is the cross-compilation seam of
  ¬ß8 #17 / probe-e, not solved here.
- **Idiomatic rendering can collide** if two used types share a short name across namespaces (e.g.
  two `Inner`s). `ToDisplayString` won't disambiguate; the safe fallback is the fully-qualified
  style. Not exercised ‚Äî the sample has no name clash. A real renderer would detect collisions and
  qualify only the conflicting names.
- **Recipe `TypeRef` is unvalidated text until resolved.** A typo'd or unknown type throws at
  resolution (with an actionable message), not at authoring. That is the same compile-gate trade the
  spike already accepts (¬ß4.6), surfaced one step earlier here.
- The probe renders **field types only** ‚Äî the one thing the naming fix is about. It does not emit
  methods, base lists, or attributes; those are other tiers.
